Q. 테스트 세트 및 학습 세트를 사용하여 
모델 개발 반복 과정을 진행하는 프로세스를 살펴보았습니다. 
각 반복 과정에서는 학습 데이터를 학습하고 
테스트 데이터로 평가하면서 테스트 데이터에 대한 평가 결과를 근거로 
학습률, 특성 등의 다양한 모델 초매개변수를 선택하고 변화를 줍니다. 
이 접근법에 잘못된 점이 있나요? 정답을 하나만 고르세요. 

A. 이 절차를 많이 반복하면 특정 테스트 세트의 특이성에 은연중에 적응하게 됩니다. 
특정 테스트 세트로 자주 평가할수록 
해당 테스트 세트에 대한 암시적 과적합이 나타날 위험이 커집니다. 
다음 과정에서 더 나은 방식을 살펴보려고 합니다.

//학습 목표
- 분할 방식에서 검증 세트의 중요성을 이해

//검증
- 데이터 세트를 학습 세트와 테스트 세트로 나누면 특정 모델이 새 데이터를 잘 처리할 수 있을 정도로 일반화되었는지 판단할 수 있음
- 그러나 초매개변수 조정을 여러 차례 실행할 때는 데이터 세트를 둘로 나누는 것만으로는 충분하지 않을 수 있음
- 따라서, 모집단에서 검증 세트를 만듦
	[         training set        ][validation set][test set]
- 학습 세트로 모델을 학습
  -> (테스트 세트가 아닌) 검증 세트로 모델을 평가
  -> 검증 세트의 결과에 따라 모델을 조정(학습률 또는 배치 크기 변경, 특성 추가 또는 삭제, 완전히 새로운 모델 설계와 같이 모델에서 가능한 모든 요소를 조정함을 의미)
  -> 검증 세트에서 가장 우수한 결과를 보이는 모델을 선택
  -> 테스트 세트의 결과를 확인
  -> 검증 세트의 결과와 테스트 세트의 결과가 일치하지 않다면, 모델이 검증 세트에 과적합할 가능성 존재

//팁
- 테스트 세트와 검증세트는 반복 사용에 따라 '마모'됨
- 즉, 초매개변수 설정 또는 기타 모델 개선을 결정할 때 같은 데이터를 더 많이 사용할수록 이러한 결과가 새로운 미지의 데이터까지 일반화될 가능성은 낮아짐
- 일반적으로 검증세트의 마모 속도는 테스트 세트보다 느림
- 따라서, 가능하면 데이터를 더 수집하여 테스트 세트와 검증세트를 '갱신'하는 것이 좋음
- 새로 시작하는 것도 좋은 재설정 방법
