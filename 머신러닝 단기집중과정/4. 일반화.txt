- 과적합에 대해 알아보기
- 모델이 적절한지 확인
- 데이터 세트를 학습 세트와 테스트 세트로 분할

//과적합: 모델이 지나치게 복잡한 것
- 이러한 과적합은 새로운 라벨이 없는 예가 들어왔을 때 문제가 될 수 있음

//개요
- 목표: 숨겨진 실제 분포에서 추출된 새로운 데이터를 제대로 예측
- 문제: 진실을 알 수 없음
  - 분포에서 추출된 샘플만 볼 수 있음

Q. 모델 h가 현재 샘플에 적합하면 다른 새로운 샘플도 잘 예측할 것이라고 신뢰할 수 있나요?

//모델이 적합한지 어떻게 알 수 있을까?
- 이론적인 측면
  - 흥미로운 분야: 일반화 이론
    - 실제론 경험적인 접근 방식을 사용하기에 위의 이론을 일단은 자세히 다루지 않음
  - 모델의 단순성/복잡성 측정 아이디어를 기반으로 함
- 직관: Occam의 면도날 원칙의 형식화
  - 모델이 덜 복잡할수록 샘플 자체의 특성을 벗어나 좋은 경험적 결과를 얻을 가능성이 높음
- 경험적인 측면
  - 질문: 모델이 새로운 데이터 샘플에 효과적으로 작동하나요?
  - 평가: 새로운 데이터 샘플을 얻습니다(테스트 세트라고 불림)
  - 모델이 테스트 세트에서 효과적으로 작동하면 다음과 같은 경우 새로운 데이터에도 잘 작동할 것이라고 추정할 수 있음
    - 테스트 세트가 충분히 큰 경우
    - 테스트 세트를 반복적으로 사용하지 않는 경우

//몇 가지 주의해야 할 점: 위의 경우 모두 기본적으로 다음 세 가지를 가정
- 분포에서 독립적이고 동일한 방식으로(i.i.d.) 임의로 예를 추출
- 분포가 정상성을 보이며 시간이 지나도 변하지 않음
  - 정상성: 데이터 세트의 데이터가 갖는 속성으로서, 하나 이상의 차원에서 데이터 분포가 일정하게 유지되는 경우. 이 차원은 대부분의 경우 시간으로서, 시간에 따라 변화하지 않는 데이터는 정상성을 갖는 것. 예를 들어 9월에서 12월까지 변화가 없는 데이터는 정상성을 가짐
- 학습, 유효성 검사 및 테스트 세트를 항상 같은 분포에서 추출

//일반화: 과적합의 위험
- 과적합 모델은 학습하는 동안 손실이 적지만 새 데이터를 잘 예측하지 못함
- 필요 이상으로 복잡한 모델을 만들면 과적합이 발생
- 머신러닝의 근본적인 과제는 데이터 적합도를 유지하는 동시에 최대한 단순화하는 것

//머신러닝의 목표는 숨겨진 실제 확률 분포에서 추출되는 새 데이터를 잘 예측하는 것
- 모델에서는 모든 데이터를 볼 수 없으며 학습 데이터 세트에서만 샘플을 추출할 수 있음

//Occam의 면도날 법칙을 머신러닝 용어로 표현하면 다음과 같음
	"ML 모델이 덜 복잡할수록 샘플의 특성 때문이 아니어도 좋은 경험적 결과를 얻을 가능성이 높습니다."

//Occam의 면도날 법칙은 통계적 학습 이론 및 컴퓨터 학습 이론 분야에서 공식화되었음
- 일반화 한계: 다음과 같은 요인을 기반으로 새 데이터에 맞게 모델이 일반화되는 정도를 통계적으로 설명하는 것
  - 모델의 복잡성
  - 학습 데이터에 대한 모델의 성능

//이론적 분석은 이상적인 가정하에 형식적인 결과를 보장하지만 실제로 적용하기 어려울 수 있음
- 당분간은 경험적 평가에 초점을 맞춰 새 데이터에 맞게 모델이 일반화되는 정도를 판단

//머신러닝의 목표는 이전에 보지 못한 새 데이터를 잘 예측하는 것
- 하지만 데이터 세트에서 모델을 만드는 경우 어떻게 이전에 보지 못한 데이터를 얻을 수 있을까?
- 따라서 한 가지 방법은 데이터 세트를 다음 두 하위 세트로 나누는 것
  - 학습 세트(Training set): 모델을 학습시키기 위한 하위 세트
  - 테스트 세트(Test set): 모델을 테스트하기 위한 하위 세트
- 테스트 세트에서 성능이 좋으면 일반적으로 다음과 같은 경우 새 데이터에서도 성능이 좋음
  - 테스트 세트가 충분히 클 때
  - 같은 테스트 세트를 반복 사용하지 않을 때

//ML 세부사항
- 일반화에서는 기본적으로 다음 세 가지 사항을 가정
  - 분포에서 독립적이고 동일하게(i.i.d.) 임의로 예를 추출. 즉, 예가 서로 영향을 미치지 않음
  - 분포가 정상성을 보임. 즉, 데이터 세트 내에서 분포가 달라지지 않음
  - 같은 분포를 따르는 부분에서 예를 추출
- 실제로는 위와 같은 가정을 위반하는 경우의 예시
  - 표시할 광고를 선택하는 모델을 고려하는 경우
    - 모델이 선택된 광고, 부분적으로 사용자가 이전에 본 광고를 기반으로 하는 경우 i.i.d. 가정을 위반
  - 1년 동안의 소매 판매 정보가 포함된 데이터 세트를 고려하는 경우
    - 사용자의 구매 패턴이 계절에 따라 변경되어 정상성을 위반

//요약
- 과적합은 모델이 새 데이터에 맞게 잘 일반화되지 않을 정도로 근접하게 학습 데이터를 적합하려고 시도하는 경우 발생
- ML의 주요 가정이 충족되지 않으면 새 데이터에 대한 예측 성능이 더 이상 이론적으로 보장되지 않음
